# 🌐 网络问题解决方案

如果遇到 `ENOTFOUND api.vercel.com` 或类似的网络连接错误，说明无法访问Vercel服务器。以下是解决方案：

## 🎯 推荐方案：使用 Railway（国内访问稳定）

**Railway 在国内访问更稳定，强烈推荐！**

### 为什么选择 Railway？

- ✅ 国内访问稳定，无需特殊网络配置
- ✅ 完全免费（每月$5免费额度）
- ✅ 支持文件写入（Vercel文件系统只读）
- ✅ GitHub推送自动部署
- ✅ 操作简单，网页操作即可

### 快速部署步骤

1. **访问 Railway**
   - 打开 https://railway.app
   - 使用GitHub账号登录

2. **创建项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 如果没有连接GitHub，先点击 "Connect GitHub" 授权

3. **选择仓库**
   - 如果代码已在GitHub，直接选择你的仓库
   - 如果代码在本地，先推送到GitHub：
     ```bash
     git init
     git add .
     git commit -m "Initial commit"
     git remote add origin https://github.com/你的用户名/你的仓库名.git
     git push -u origin main
     ```

4. **自动部署**
   - Railway会自动检测Node.js项目
   - 自动安装依赖并启动服务
   - 等待部署完成（约1-2分钟）

5. **获取访问地址**
   - 部署完成后，点击项目
   - 在 "Settings" -> "Networking" 中可以看到公网URL
   - 或者点击 "Generate Domain" 生成一个域名

6. **完成！**
   - 现在你的服务已经在线了，可以访问了！

## 🔧 Vercel 网络问题解决方案

如果你坚持使用Vercel，可以尝试以下方法：

### 方法1：使用Vercel网页版（推荐）

**无需命令行，直接在网页上操作！**

1. **访问 Vercel 网站**
   - 打开 https://vercel.com
   - 使用GitHub账号登录

2. **导入项目**
   - 点击右上角 "Add New..." -> "Project"
   - 选择 "Import Git Repository"
   - 如果没有看到你的仓库，点击 "Adjust GitHub App Permissions" 授权GitHub

3. **配置项目**
   - 选择你的代码仓库
   - Framework Preset: 选择 **"Other"**
   - Root Directory: 留空（使用根目录）
   - Build Command: 留空（不需要构建）
   - Output Directory: 留空
   - Install Command: `npm install`
   - Development Command: 留空
   - **重要**：在 "Override" 中设置：
     - Start Command: `node server.js`

4. **部署**
   - 点击 "Deploy"
   - 等待部署完成（约1-2分钟）

5. **完成！**
   - 部署完成后会显示一个URL，例如：`https://your-project.vercel.app`
   - 这个URL就是你的服务地址

**优点**：完全不需要命令行，避免了网络连接问题！

---

### 方法2：配置代理（高级）

如果你有代理工具，可以配置代理：

#### Windows (使用系统代理)

1. 设置系统代理
2. 在命令行中设置代理环境变量：
   ```bash
   set HTTP_PROXY=http://127.0.0.1:7890
   set HTTPS_PROXY=http://127.0.0.1:7890
   ```
   （将7890替换为你的代理端口）

3. 然后运行：
   ```bash
   vercel login
   vercel
   ```

#### 使用 npm 代理

```bash
npm config set proxy http://127.0.0.1:7890
npm config set https-proxy http://127.0.0.1:7890
```

### 方法3：使用其他平台

如果Vercel无法访问，可以使用以下替代平台：

#### Render（稳定可靠）

1. 访问 https://render.com
2. 使用GitHub账号登录
3. 点击 "New" -> "Web Service"
4. 选择你的GitHub仓库
5. 配置：
   - Build Command: `npm install`
   - Start Command: `npm start`
6. 点击 "Create Web Service"

**优点**：国内访问稳定，免费套餐可用
